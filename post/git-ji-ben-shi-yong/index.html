git基本使用

2020-02-27

git

创建仓库

在任意目录下使用

git init

将这个目录变成git可管理的路径即git仓库，

并且会在里面多一个.git文件夹

将需要被管理的文件添加入此目录，使用

git add  XXXX.XX

将告诉git，把文件添加入仓库，使用

git commit -m "本次提交说明"

将文件提交

（一次可以add添加多次，然后commit提交一次）

这样，该文件就存在于git的版本库中。

使用

git status

可以查看当前仓库的状态，是否处于修改未添加，或者添加未提交还是其他状态。

通过

git diff

可以查看当前修改了什么

通过

git log

可以查看历史记录

返回历史版本

在git中用head表示当前版本，需要返回上个版本就是head,上上个版本就是head^，使用

git reset --hard head^

即可返回上个版本

重新回到未来的版本

git reset --hard xxxx(commit id)

可以通过以下命令

git reflog

来查看记录的每一条命令，从而获得commit id

撤销修改

当想抛弃工作区的修改，使用

git checkout --file

就可丢弃（未添加到git仓库时），当修改了并且Git add到暂存区时，使用

git reset HEAD <file>

就可以把暂存区的修改回退到工作区，然后通过上一步将工作区修改丢弃

删除文件

如果通过直接删除或者rm命令删除，这时候git知道删除了文件，所以工作区和版本库就不一致了，要么通过在版本库中彻底删除，通过

git rm xxx   //删除
git commit

要么恢复，通过

git checkout --xxx

远程仓库绑定

在GitHub中创建一个空仓库，要将本地和远程仓库连起来，实现同步，通过下面命令将本地仓库和远程仓库关联

git remote add origin git@github.com:用户名/远程仓库名.git

然后就可以把本地仓库的内容推到远程仓库里去

通过

git push -u origin master

将当前分支推送到远程
