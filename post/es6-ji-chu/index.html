二、字符串扩展

字符的Unicode表示法

\uxxxxxx形式表示一个字符，xxxxxx是这个字符的Unicode码，范围为\u0000~\uFFFF之间的字符。超出这个范围的字符，必须用两个双字节的形式表达

“\uD842\uDFB7” // "吉"

**ES6中，进码放进大括号就能正确读出这个字符 **

”\u{20BB7}“ // "吉"

”\u{41}\u{42}\u{43} " // "ABC"

codePointAt()

在JavaScript内部，字符是以utf-16的格式储存，每个字符固定为两个字节，Unicode大于0xFFFF的字符会认为是两个字符，即四个字节，codePointAt()会正确返回该字符的码,如果是常规的两字节储存的字符，codePointAt()和方法charCodeAt()结果相同。

var s= '吉a';
s.length                    // 3(吉占两个字符)
s.codePointAt(0)          	// 134071
s.codePointAt(1)		   // 57271
s.codePointAt(2)            // 97

顺便，codePointAt()是最简单的判断一个字符是两个字节还是四个字节。

function is32bit(a)
{
	return a.codePointAt(0)>0xFFFF;
}
is32bit("吉")             //true
is32bit("a")              //false

String.fromCodePoint()

作用与codePointArt()相反，将Unicode码转成字符，如果Sring,formCodePoint()有多个参数，则将会将字符拼成字符串返回。

String.formCodePoint(9x20BB7)                   // "吉"
String.formCodePoint(0x78,0x1f680,0x79)         // true

箭头函数注意事项

  1. 函数体内的this对象就是定义时所在的对象，而不是使用时所在的对象。

  2. 不可以作为构造函数。

  3. 不可以使用argument对象。

  4. 不可以使用yield命令

    注意

    其中，箭头函数没有this,所以绑定的是外层的代码块的this，由于这一点，可以用箭头函数来绑定this。

    箭头函数也可以嵌套使用

尾调优化

尾调是指函数的最后一步调用另一个函数

function f(x){
	return g(x);
}

尾调中不能出现其他运算（赋值，算术运算都不能算尾调）

尾递归是在函数的最后一步调用自身，

由于递归需要同时保存上千的调用帧，很容易出现栈溢出，但尾递归只存在一个调用帧，不存在栈溢出的现象，但可以用循环代替递归来实现递归进行优化
